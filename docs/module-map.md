## TOC load order

`LootHistory.toc` orders files by dependency, not alphabetically. Libraries first, then `core/` (Compat first), defaults, locales, settings, modules (Attribution and Filters before Collector).

1. `libs/` ‚Äî vendored LibStub, CallbackHandler-1.0, AceAddon-3.0, AceEvent-3.0, AceTimer-3.0, AceConsole-3.0, AceDB-3.0, AceGUI-3.0, LibSharedMedia-3.0, LibDataBroker-1.1, LibDBIcon-1.0. Committed in-tree (Ka0s Standard v2.0.0 ‚Äî vendoring is mandatory; `.pkgmeta` externals are forbidden). Don't edit.
2. `core/Compat.lua` ‚Äî **loads first** of `core/`. Hangs `NS.Compat` (the private namespace `NS` is the second file vararg, not a global) and populates every deprecated/varying API shim: `GetPlayerMapID`, `GetActiveKeystoneLevel`, `HookUseContainerItem` / `ContainerItemHasLoot`, `HookGetQuestReward` / `CurrentQuestID`, `IsSpellTargeting`, `GetSpellName`, `GetMailHeader` / `IsAuctionHouseMail`, `GetZone`, `DecodeGUID` (+ `UNIT_KINDS`), `QualityFromLink` / `QualityLabel`, `GetItemInfo`, `ScanBound`, `GetItemExtras`, `GetAddOnMetadata`. Each is gated by a direct `C_*`/global presence check (`Compat.lua:5-7`) ‚Äî no `WOW_PROJECT_ID` branching ‚Äî so anything later can rely on `NS.Compat` existing.
3. `core/Constants.lua` ‚Äî exposes `NS.Constants` (alias `NS.C`) with the `SourceType` enum (stable string keys = the export contract; `Constants.lua:8-13`), `SourceOrder` / `SourceLabel`, the `SOURCE_IMPLEMENTED` mute-UI gate (`Constants.lua:33-37` ‚Äî `CRAFT`/`ROLL` are enum'd but unstamped, so hidden), `ITEMCLASS_QUEST` (locale-independent quest-item class, `Constants.lua:44`), `CONTEXT_TTL` (1.5 s), the quality/retention/source option tables, and `FONT_MONO`. Publishes `NS.SourceType` / `NS.Confidence` aliases.
4. `core/Namespace.lua` ‚Äî sets `NS.name`, `NS.version` (`"1.1.0"`, `Namespace.lua:5`), and `NS.PREFIX` (the cyan `|cff00ffff[LH]|r` chat tag, slash-commands-¬ß4). Modules still self-publish idempotently; nothing else wired here.
5. `core/State.lua` ‚Äî `NS.State`, the runtime-only mutable state: `lootContext` (the single-slot attribution stamp), `encounter` / `keystone` (instance context), the session flags `cleanupDone`, `debug`, `testRecords`, and `viaWhitelistIDs` (the derived whitelist-orphan index for `VisibleHistory`, issue #14). Reset every load ‚Äî never persisted to SavedVariables.
6. `core/Util.lua` ‚Äî pure helpers: `PlayerKey` (`Name-Realm`), `SplitPath`, `FormatClock` / `FormatDate` / `FormatMoney` / `FormatBytes`, `RangeFrom` (date-range key ‚Üí `from` epoch, used by the shared filter bar's Date dropdown), and `BuildLootPatterns` / `ParseSelfLoot` (compile the localized loot global-strings once and parse a `CHAT_MSG_LOOT` line into link + quantity). Also the shared **secret-safe chat printer** ‚Äî `NS.Print` (`= NS.Util.print`) plus `IsConcatSafe` / `SafeToString`: every module does `local print = NS.Print` and emits `print("msg")`, so the cyan `NS.PREFIX` tag is prepended once and each arg is secret-stringified (events-frames-taint-¬ß8). `NS.Print` is reclaimed from AceConsole's `:Print` mixin in `core/LootHistory.lua`.
7. `core/LootHistory.lua` ‚Äî the AceAddon bootstrap. `AceAddon:NewAddon(NS, addonName, "AceEvent-3.0", "AceTimer-3.0", "AceConsole-3.0")` promotes `NS` in place (no `_G.LootHistory` rebind ‚Äî the namespace stays private) and sets `NS.addon` / `NS.bus`. Defines `NS.NewBusTarget()` ‚Äî the fresh-AceEvent-target factory every message consumer must use so two consumers can't clobber each other on the shared bus (`LootHistory.lua:15-26`; see [message-bus.md](message-bus.md)). Defines the three lifecycle hooks (below).
8. `core/Database.lua` ‚Äî defines `NS:InitDB` (`AceDB-3.0:New("LootHistoryDB", NS.defaults, true)` ‚Äî account-wide) and `NS:RunMigrations` (reads/writes `db.global.schemaVersion`; a no-op seam today, `Database.lua:13-18`), then `NS.Database` with the read/write path: `History` / `VisibleHistory` (blacklist-aware; hides blacklisted ids without deleting) / `ActiveHistory` (the test-mode-aware read seam behind Query/Stats), `Count`, `Add` (fires `RecordAdded`), `QueryList` / `Query`, `Export` (the v2 export contract), `Stats` (the single O(n) pass all Insights widgets consume), `DeleteAt` / `Delete` / `Purge` / `PruneOld` (all fire `HistoryChanged`), and `StorageStats`. Does not create the DB at file-load.
9. `defaults/Global.lua` ‚Äî `NS.defaults.global`: `schemaVersion = 1`, `history = {}`, `blacklist = {}` / `whitelist = {}` (the item-id filter lists), the `settings` table (enabled, qualityThreshold, excludeQuestItems, excludedSources, retentionDays, windowScale, window), and `minimap = { hide = false }`. History, lists and settings all live under `.global` (account-wide); debug is deliberately NOT here (session-only). See [data-model.md](data-model.md).
10. `locales/enUS.lua` ‚Äî `NS.L` with a metatable that returns the key itself, so English strings work untranslated and missing keys never error. The addon ships English-only: no user-facing string routes through `NS.L` yet (an accepted scope decision, `enUS.lua:7-11`); the seam is kept for a future localization pass.
11. `settings/Schema.lua` ‚Äî `NS.Schema` (alias `S`): one row per setting (`S.Schema`, `Schema.lua:10-64`) driving AceDB defaults, panel widgets, and the slash CLI. `S:Set(path, value)` is the single user-setting write seam ‚Äî validate ‚Üí deep-copy ‚Üí write to `NS.db.global` ‚Üí `onChange` (`Schema.lua:109-117`); the deep-copy stops a reset from aliasing the DB to a shared default table. Also owns `NS.COMMANDS`, the slash dispatch/help table (`Schema.lua:142-174`). `debug` is intentionally not a Schema row.
12. `settings/Slash.lua` ‚Äî `NS.Slash`. Registers `/lh` and `/loothistory` via AceConsole (`Slash.lua:36-39`), dispatches the verb against `NS.COMMANDS`, and generates the help index from it (bare `/lh` prints help ‚Äî Ka0s slash-commands-¬ß4). Provides the schema-driven `CliGet` / `CliSet` / `CliList` (via the pure `BuildListLines`, with the shared `FormatSchemaValue` / `FormatKV` helpers) / `CliReset` / `CliResetAll` / `CliVersion`, the `ResetEverything` full wipe, and registers the purge / reset-all `StaticPopupDialogs` confirm dialogs at file-load (`Slash.lua:7-27`). See [slash-dispatch.md](slash-dispatch.md).
13. `settings/Panel.lua` ‚Äî `NS.Panel`. Registers the parent `Settings.RegisterCanvasLayoutCategory` (a logo + tagline + slash-command landing page) plus two subcategories: `General` (the schema settings) and `Filters` (blacklist/whitelist item-id management ‚Äî a custom non-schema page: add-by-id/link box + a live list with Remove per id, driven by `NS.Filters`). Renders `NS.Schema.Schema` into a lazy two-column AceGUI grid (checkbox/dropdown/slider/multi-check makers), a live History stats + Purge section, and the always-shown-scrollbar patch (Ka0s options-ui-¬ß10). Writes route through `NS.Schema:Set` (schema rows) or `NS.Filters` (the lists); `P:Open` is combat-gated. See [settings-panel.md](settings-panel.md).
14. `modules/Attribution.lua` ‚Äî **loads before Filters/Collector.** The source-resolution engine (`NS.Attribution`). `Stamp` writes the single-slot `State.lootContext` (fresh for `CONTEXT_TTL`, not cleared on consume ‚Äî one loot window emits many lines); `Consume` reads it or falls back to `OTHER`/`INFERRED`. `ResolveLootSource` maps a loot-slot GUID + instance state ‚Üí source + detail (Creature‚ÜíKILL, GameObject‚ÜíMPLUS/CONTAINER, Item‚ÜíCONTAINER). `Attribution:Enable` (called from `OnEnable`, never at file-load) registers the peripheral stampers: LOOT_OPENED, ENCOUNTER/CHALLENGE_MODE, TRADE_ACCEPT_UPDATE, QUEST_TURNED_IN, a player-only `RegisterUnitEvent` frame for deconstruct spell-success, and `hooksecurefunc` hooks for vendor buy / mail take / container use / quest reward. See [attribution.md](attribution.md).
15. `modules/Filters.lua` ‚Äî `NS.Filters`, the blacklist/whitelist item-id lists (issue #14). **Loads before Collector.** Copy-on-write `AddBlacklist` / `AddWhitelist` (each drops the id from the sibling list) / `RemoveBlacklist` / `RemoveWhitelist` write a fresh table to `NS.db.global.{blacklist,whitelist}` and fire `SettingsChanged` (the Collector re-caches) + `HistoryChanged` (the browser re-queries); `IsBlacklisted` / `IsWhitelisted` / `SortedIDs` / `ParseItemID` (id from a number, link, or itemString) support the panel + right-click UIs. Data-only ‚Äî no `Enable`. An architecture-¬ß5 carve-out from `Schema:Set`.
16. `modules/Collector.lua` ‚Äî `NS.Collector`, the acquisition path. `ShouldRecord` is the pure gate: a blacklisted id is an absolute veto; otherwise the item records if it passes the normal gate (quality / excluded-source / quest-item, keyed on the locale-independent `ITEMCLASS_QUEST`), or ‚Äî failing that ‚Äî if it is whitelisted, in which case it returns `true, "whitelist"` so the row is stamped `viaWhitelist` (letting an un-whitelist hide it later). `BuildRecord` assembles the record. `OnChatMsgLoot` parses the self-loot line, consumes the attribution context, gates (passing the item id + the cached lists), gathers item/zone extras, sets `viaWhitelist`, and calls `Database:Add`. `Collector:Enable` registers `CHAT_MSG_LOOT` and ‚Äî on its own `NS.NewBusTarget()` ‚Äî `Ka0s_LootHistory_SettingsChanged`, refreshing the hot-path upvalues (`enabled`/`qualityThreshold`/`excludedSources`/`excludeQuestItems`/`blacklist`/`whitelist`) rather than reading the DB per loot (Ka0s events-frames-taint-¬ß7).
17. `modules/Browser.lua` ‚Äî `NS.Browser`, the standalone window (Ka0s standalone-windows reference implementation). Owns the non-secure `LootHistoryWindow` frame (HIGH strata, movable/resizable, ESC via `UISpecialFrames`), the flat `SKIN` + `ApplySkin` re-skin seam, window position/scale persistence to `NS.db.global.settings.window` (an architecture-¬ß5 carve-out from `Schema:Set`), the History/Insights tab strip (lazy pane build), the **shared singleton filter bar + footer** (multi-select dropdowns Bound / Quality / Type / SubType / Source / Zone / Character; `CurrentFilter` feeds both the table and the Insights charts ‚Äî issue #13), the saved-view logic and session-only current-player character scope, the **tab-aware `Export` button** (`OpenExport` ‚Üí loot rows on History, analytics on Insights ‚Äî issue #15), and the LibDataBroker launcher + LibDBIcon minimap button. Exposes `MakeDropdown` / `GetWindow` for `Export` to reuse. `Browser:Enable` subscribes to `SettingsChanged` / `HistoryChanged` / `RecordAdded` on its own bus target and calls `SetupMinimap`. See [browser.md](browser.md).
18. `modules/BrowserTable.lua` ‚Äî `NS.BrowserTable`, the virtualized pooled-row History table. Defines the `COLUMNS` model (Character always last; SubType after Type), the filter‚Üísort‚Üígroup‚Üíslice‚Üíbind pipeline over `Database:QueryList(CurrentRecords, filter)`, `OrderedFilteredRecords` (the on-screen order the Export Current-View dataset serializes), a `FauxScrollFrame` + row pool (never one frame per record, Ka0s standalone-windows), collapsible group headers, the class-icon / bound-lock markup, the right-click row menu (link / **blacklist** / delete), and the deterministic-PRNG synthetic dataset for `/lh test` (`ToggleTestMode` publishes `NS.State.testRecords`, so both the table and Insights render off the same fake data).
19. `modules/Export.lua` ‚Äî `NS.Export`, the export modal (`Export:Open`). Builds a small skinned window with a Data Set dropdown (All Data / Current View) and Export-to-CSV / Export-to-AI (stub) buttons, reusing `Browser:MakeDropdown` / `ApplySkin` / `MakeCloseButton` and centering on the History window. Driven by a per-open `config = { title, providers, csv }` ‚Äî the invoking tab supplies the window header ("Export History" / "Export Insights"): `CSV` serializes loot rows (19 record fields + derived `date`/`time`/human `quality`/`sellPrice` beside `*Raw` columns + `wowheadLink`) for the History tab; `InsightsCSV` serializes a `Database:Stats` result into a sectioned analytics CSV for the Insights tab (issue #15). `WowheadLink` builds a `wowhead.com/item=‚Ä¶?bonus=‚Ä¶` URL from the item's bonus IDs; output opens in Export's own copy window. Called directly by the Browser ‚Äî no bus message.
20. `modules/Analytics.lua` ‚Äî `NS.Analytics`, the Insights tab. Attaches to the Browser's Insights pane, builds stat/highlight cards + a stack of breakdown sections (source, vendor value, quality, quality mix, item type, bound type, per-character, per-day + value-per-day strips, hour-of-day, weekday, M+ keystone, confidence) + top zones/items/value lists, all off a single `Database:Stats(filter)` pass **scoped by the browser's shared filter** (`Browser:CurrentFilter`, issue #13 ‚Äî no range selector of its own). Widgets are pooled and re-laid-out on resize; `BuildCharts` subscribes to `RecordAdded` / `HistoryChanged` on its own bus target for live refresh.
21. `modules/DebugLog.lua` ‚Äî **loads last** of `modules/`. `NS.DebugLog`, the session-only debug console (`LootHistoryDebugWindow`, DIALOG strata, styled by `Browser:ApplySkin`) with a `ScrollingMessageFrame` log in JetBrains Mono, Copy/Clear buttons, and a header `Debug: ON/OFF` toggle. Defines the global sink `NS.Debug(tag, fmt, ‚Ä¶)` (`DebugLog.lua:248-262`) ‚Äî zero-allocation no-op when `NS.State.debug` is off, otherwise appends a tagged `<ts> | [<tag>] <content>` line. `D:SetEnabled` is the single state seam shared by `/lh debug on|off` and the header toggle. It loads after the other modules only reference `NS.Debug` at runtime inside `NS.State.debug` guards, so the late definition never matters.

## AceAddon lifecycle

1. **TOC file-load** as listed above. After load: `NS.Compat` / `NS.Constants` / `NS.State` / `NS.Util` are populated; `core/LootHistory.lua` has run `AceAddon-3.0:NewAddon` to promote `NS` in place (setting `NS.addon` / `NS.bus` and defining `NS.NewBusTarget`); `NS.defaults` is set; `NS.L` exists; `NS.Schema` (with `NS.COMMANDS`), `NS.Slash` (with its confirm dialogs registered) and `NS.Panel` exist; each module has published `NS.X = NS.X or {}` and defined its methods but **not** enabled ‚Äî no module touches a WoW event/hook/frame API at file-load (so the headless test harness can load every file).
2. **`addon:OnInitialize`** (Ace lifecycle, fires on `ADDON_LOADED`; `LootHistory.lua:28-35`): registers the JetBrains Mono font with LibSharedMedia; `NS:InitDB` builds the AceDB instance and runs `NS:RunMigrations` (no-op at `schemaVersion = 1`; the seam future schema changes hook into); `NS.Schema:Register` runs boot validation (every schema path must resolve against the defaults); `NS.Slash:Register` binds `/lh` + `/loothistory`; `NS.Panel:Register` registers the Blizzard canvas categories (deferred content ‚Äî panels render lazily on first `OnShow`).
3. **`addon:OnEnable`** (`LootHistory.lua:37-45`): registers `PLAYER_ENTERING_WORLD` ‚Üí `OnEnterWorld`, then enables the modules in order ‚Äî `Attribution:Enable` (peripheral event registrations + read-side hooks), `Collector:Enable` (`CHAT_MSG_LOOT` + the `SettingsChanged` upvalue-refresh subscription), and `Browser:Enable` (bus subscriptions + `SetupMinimap`). `Filters` is data-only (no `Enable`); `BrowserTable` and `Analytics` have **no** `Enable` ‚Äî they attach lazily when the Browser first builds their pane (`Analytics` subscribes to the bus in `BuildCharts`); `DebugLog` likewise builds its frame lazily. `Panel` and `Slash` do their work in `OnInitialize`, not here.
4. **`addon:OnEnterWorld`** (`LootHistory.lua:48-56`): guarded to run once per session (`NS.State.cleanupDone`). Schedules `Database:PruneOld` via `C_Timer.After(5, ‚Ä¶)` so retention cleanup lands 5 s after the login/zone spike rather than during it. `PruneOld` drops records older than `settings.retentionDays` (0 = keep Always) by rebuild-and-swap and fires `HistoryChanged`.
